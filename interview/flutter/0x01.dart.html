                                # ❓final 与 const 区别？

                                💡
                                • const : 是编译时常量，必须初始化。
                                • final : 运行时常量，支持惰性初始化，只能被赋值一次的变量。
                                final date = DateTime().now(); // ✅
const date = DateTime().now(); // ❌ 编译错误

                                # ❓Dart 中 .. 表示什么意思

                                💡

                                在Dart语言中，..被称为级联运算符 (Cascade notation)。

                                级联运算符允许你在同一个对象上执行一系列操作。这可以使你的代码更加简洁，因为你只需要引用对象一次，然后执行多个操作。

                                例如，如果你有一个对象并且想在该对象上调用多个方法，而没有级联操作符，你将需要编写：

                                var myObject = SomeClass();
myObject.method1();
myObject.method2();
myObject.method3();

                                使用级联操作符，你可以简化为：

                                var myObject = SomeClass()
  ..method1()
  ..method2()
  ..method3();

                                在这个例子中，..运算符允许我们在myObject上连续调用多个方法，而无需每次都引用myObject。

                                值得注意的是，级联操作的结果是表达式的目标，也就是说依然是myObject，而不是最后一条语句的返回值。

                                # ❓Dart 的作用域

                                💡

                                在Dart中，作用域是一种规定变量，函数和对象的可访问范围的机制。换句话说，作用域规定了在何处可以访问和修改特定标识符。以下是Dart的一些作用域规则：

                                1. 局部作用域：在函数或方法中声明的变量只能在其内部使用。这个区域的代码对该变量有全权访问，包括读取和修改。
                                void main() {
  var x = 'Hello World'; // x is only accessible within main() function
  print(x);
}
                                1. 全局作用域：在函数或方法外部声明的变量可以在程序的任意位置使用。这样的变量通常称为全局变量。
                                var x = 'Hello World';  // x is accessible anywhere in the code

void main() {
  print(x);  // can access x within main() function
}
                                1. 词法作用域：Dart是词法作用域的语言，这意味着变量的作用范围是静态的，由代码的布局决定。
                                var x = 'Hello World';  // x is accessible anywhere in the code

void main() {
  var x = 'Hello Dart';  // This x variable is not the same as the global x
  print(x);  // This will print 'Hello Dart'
}

                                在上述代码中，main()函数内部的x和全局的x是两个不同的变量。尽管它们有相同的名称，但是由于它们在不同的作用域中，因此它们是不同的。

                                1. 块作用域：在Dart中，大括号 {} 内部声明的变量具有块级作用域，也就是它们仅在这个块中可见。
                                void main() {
  if(true){
    var x = 'Hello';  // x is only accessible within this if block
    print(x);  // This will print 'Hello'
  }
  print(x);  // Error: x is not accessible outside the if block
}

                                以上就是Dart中关于作用域的一些基本规则。了解这些规则在编写Dart代码时非常重要，因为它们会影响变量在何处可见，如何命名，以及何时生效和失效。

                                # ❓说一下 mixin

                                💡

                                在Dart中，mixin是一种在多个类之间重用代码的方式。它提供了一种实现方法重用的方式，并且不需要创建和使用复杂的类层次。

                                你可以创建一个mixin，然后将其"混入"另一个类中，基类可以通过这种方式来使用mixin中定义的变量和方法。mixins的引入让我们避免了因继承而导致的代码冗余和复杂性。

                                以下是一个基本的mixin示例：

                                mixin Flyable {
  fly() {
    print('I can fly!');
  }
}

class Bird with Flyable { }

void main() {
  Bird b = Bird();
  b.fly();  // 打印 'I can fly!'
}

                                在上面的例子中，Flyable就是一个mixin，它定义了一个fly的方法，然后我们在Bird类中使用with关键字混入了Flyable，接着我们就可以在Bird类的实例上调用fly方法，正如我们在Bird类中直接实现了fly方法一样。

                                另外需要注意的是，一个类可以混入多个mixin，mixin也可以继承其它mixin。

                                mixin存在一些限制和规则。例如，只有当一个类不定义构造函数，且它自身不是从类扩展而来时，它才可以被用作mixin。这是因为mixin本质上是一种对接口的补充，它描述的是行为，而不是状态。

                                # ❓mixin extends implement 之间的关系

                                💡

                                在Dart中，mixin, extends, 和 implements 关键字提供了不同的方式去继承和复用代码。它们之间的区别如下：

                                • extends：在Dart中，类可以通过extends关键字继承一个父类。但是Dart不支持多继承，一个子类只能继承一个父类。

                                  class Vehicle {
  void drive() {
    print('Driving...');
  }
}

class Car extends Vehicle {
  void honk() {
    print('Honking...');
  }
  @override
  void drive() {
    print('Driving car...');
  }
}

                                • implements：如果你需要继承一个类，但是你需要覆写所有的方法和变量，你可以使用implements关键字。它强制您覆盖实现所有父类的成员。

                                  class Vehicle {
  void drive() {
    print('Driving...');
  }
}

class Car implements Vehicle {
  void drive() {
    print('Driving car...');
  }
}

                                • mixin：在Dart中，mixin是一种将类的代码在多个类层次结构中复用的方法。使用mixin关键字定义类，然后使用with关键字将其混入另一个类。注意，Mixin类不能被实例化，也不能有构造函数。

                                  mixin Musical {
  void sing() {
    print('Singing...');
  }
}

class Musician extends Person with Musical {
  // We can now use the sing() method in Musician.
}

                                由此可见，mixin, extends, 和 implements 在对象继承和行为复用方面提供了多种策略，具体使用哪种方式取决于你的面向对象设计需求。

                                # ❓Dart 为啥没有反射

                                💡

                                因为反射会影响 Dart 的 tree shaking(摇树优化)。这是一种无用代码裁剪技术，有利于优化发布版本的 App 体积。如果反射机制存在，会存在隐式代码，tree shaking 时无法准确定位运行时哪些代码不会被调用，冗余代码就没法被裁剪。

                                Dart其实是支持反射的，反射在 dart:mirrors库中实现，提供了一系列用于执行反射操作的类和方法。

                                然而，反射在某些Dart的使用场景下并不可用。

                                在Flutter中，为了优化app的大小和性能，Dart的反射库dart:mirrors并没有被包含进来。因为反射会增加应用程序的体积，同时反射在运行时对程序类型和结构的查询修改也会消耗更多资源从而对性能带来影响。

                                在Web上，用Dart编译到JavaScript的时候，也是不支持反射的。这是由于在转译到JavaScript时，为了使生成的JavaScript尽可能的小和快，也不支持反射。

                                所以虽然Dart语言本身是支持反射的，但因为反射对代码体积和性能的消耗，在部分场景下Dart的反射能力被刻意禁用了。

                                # ❓Dart 中如何使用反射?

                                💡

                                在Dart中，反射是一种动态语言特性，可以让你在运行时检查和修改你的程序。反射现在在dart:mirrors包中实现，该包提供了一系列用于执行反射操作的类和方法。

                                • ClassMirror : 类
                                • InstanceMirror : 实例
                                • ClosureMirror : 闭包
                                • DeclarationMirror : 类属性
                                • IsolateMirror : Isolate
                                • MethodMirror : 方法

                                以下是一个反射的简单示例，使用dart:mirrors的reflect和Symbol去实例化类和调用其方法：

                                import 'dart:mirrors';

class MyClass {
  MyClass(String message) {
    print('New object created. Message: $message');
  }

  void myMethod(String message) {
    print('myMethod called. Message: $message');
  }
}

void main() {
  // 获取 MyClass 的类镜像
  ClassMirror classMirror = reflectClass(MyClass);

  // 用 MyClass 的构造函数创建新实例
  var myClassInstanceMirror = classMirror.newInstance(Symbol(''), ['Hello Constructor']);
  
  // 获取 MyClass 的实例
  MyClass myClassInstance = myClassInstanceMirror.reflectee;

  // 获取 MyClass.myMethod 的方法镜像
  var myMethodMirror = reflect(myClassInstance).type.instanceMembers[Symbol('myMethod')];

  // 通过反射调用 myMethod
  myMethodMirror.invoke(myClassInstanceMirror, ['Hello Method']);
}

                                在上述代码中，创建了MyClass类的实例并且调用了myMethod方法。所有这些都是通过反射在运行时动态进行的。

                                请注意，dart:mirrors库（以及所有的反射在Dart中）在web场景中并不被所有的编译目标(例如dart2js或flutter web)支持，并且可能对程序的体积和性能有所影响。因此，在将你的代码部署到生产环境之前，应仔细考虑是否需要使用反射。

                                # ❓说一下 Future

                                💡

                                在Dart中，Future是一个代表未完成的计算的类。你可以把它想象成一个会在将来某个时间返回一个值（或错误）的操作。

                                对于那些可能需要一些时间才能完成的操作（比如文件读写、网络请求等），Future提供了一种优雅的方式来处理最后的结果和错误，而无需阻塞程序的主线程。

                                Future通常有两种状态：未完成和完成。在Future完成之前，你可以添加一些回调，以在Future完成时执行。一旦Future完成，它会持有一个值（作为计算结果）或一个错误。

                                在Dart中，你可以使用Future.then添加回调，也可以使用async和await来简化异步操作。async和await关键字可以让你用类似于同步代码的方式来书写和理解异步代码。

                                以下是一个简单的使用Future的例子：

                                void main() {
  // 创建一个 Future
  Future<String> future = Future.delayed(
    Duration(seconds: 2),
    () => 'Hello, Future!',
  );
  
  // 注册回调
  future.then(print);  // 当 Future 完成时，会打印 'Hello, Future!'
}

                                这个例子创建了一个2秒后完成并返回'Hello, Future!'的Future，并且使用Future.then添加了一个回调。当Future完成，并返回'Hello, Future!'时，会调用print函数。

                                这只是Future的基本用法。实际使用时，你可能也会遇到链式调用、错误处理、并发操作等更复杂的情况，Future类提供了包括catchError、whenComplete、wait等在内的多种方法来处理这些情况。

                                # ❓说一下 Stream

                                💡

                                在Dart中，Stream用于序列化异步事件。与只完成一次的Future不同，Stream能够发出多个值。

                                你可以把Stream理解成一个异步的数组，就像一个流水线，你可以在流水线的一端投入数据，在另一端处理数据，数据在流水线上移动时并不会阻塁主线程的执行。

                                以下是一个使用Stream的例子：

                                void main() {
  // 创建一个 Stream 
  const data = [1, 2, 3, 4, 5];
  final stream = Stream.fromIterable(data);

  // 监听 Stream
  stream.listen(print);  // 打印 1, 2, 3, 4, 5
}

                                在这个例子中，我们创建了一个从数据源data中发出值的Stream，然后通过listen()方法来设置值的处理函数，每当有新值发出时，都会调用print函数打印这个值。此例子中，将会打印出[1, 2, 3, 4, 5]。

                                你可以使用Stream来处理一次或多次的异步事件，比如文件或数据库操作的结果、网络请求的响应、定时器事件等。而向Stream添加数据的最常用方式是使用StreamController，你可以使用StreamController管理数据的添加和Stream的监听。

                                另外，Dart的Stream还提供了更为强大的功能，比如你可以使用map、where等下载数据直接对流进行操作，或者使用async/await for关键字来简化Stream的处理。

                                # ❓await for 如何使用

                                💡

                                在Dart中，await for循环用于异步迭代 Stream 的元素。

                                当你使用await for，你的代码会在每个元素准备就绪时被暂停并恢复执行，而不是立即获取所有元素。这对于处理大量数据，或者处理可能永远不会结束的流（例如，常开的网络连接）特别有用。

                                以下是一个使用await for的例子：

                                Future<void> printStream(Stream<int> stream) async {
  await for (var number in stream) {
    print(number);
  }
}

void main() {
  var stream = Stream.fromIterable([1, 2, 3, 4, 5]);
  printStream(stream);
}

                                在这个例子中，printStream函数会异步等待并打印流stream中的每个数字。

                                要注意的是，只有在异步函数中（函数声明包含async关键字）才能使用await for。并且，如果流发送错误，await for循环会抛出异常并立即终止。如果你想处理可能的错误，你应该在await for循环中使用try/catch。

                                还有一点要注意，使用await for会暂停函数的执行，直到整个Stream被处理完毕。如果你还想处理其他的异步操作，或者你不希望等待整个Stream处理完毕，你应该考虑使用Stream的listen方法监听流的处理过程。
